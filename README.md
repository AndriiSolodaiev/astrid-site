# Ініціалізація

В терміналі виконати наступну команду:

```
npx create-website-template
```

Після цього потрібно ввести назву проекту в терміналі.

Нова папка з проектом створиться в тому місці де була виконана команда.

# Template Project

Цей проект є стартовим шаблоном для розробки фронтенду. Архітектура базується на методології **Feature-Sliced Design (FSD)**, яка була адаптована та спрощена під специфіку наших задач.

## 🏗 Архітектура проекту

Проект розділений на шари (layers). Дотримання ієрархії імпортів є критично важливим для уникнення циклічних залежностей та збереження модульності.

### Структура папок (`src/`)

Основна розробка ведеться в чотирьох директоріях:

1.  **`app/`**
    - Тут міститься загальна логіка, ініціалізація, глобальні стилі та конфігурації, які доступні всьому проекту.
    - Це "корінь" додатку, який збирає все разом.

2.  **`pages/`**
    - Містить **унікальні сторінки** додатку (наприклад, `home`, `about`, `contacts`).
    - Компоненти тут **не перевикористовуються**.
    - Кожна сторінка зазвичай має свою точку входу (JS файл) та основний файл розмітки (Pug).
    - **Правила імпорту:** Може імпортувати з `@widgets` та `@shared`.

3.  **`widgets/`**
    - Містить **складні самодостатні компоненти** (наприклад, `Header`, `Footer`, `ContactForm`, `MapSection`).
    - Віджет — це повноцінний блок, який має свою JS логіку, Pug-розмітку, стилі (SCSS) та, за необхідності, свої картинки/асети.
    - Віджети створені для **перевикористання** на різних сторінках.
    - **Правила імпорту:** Може імпортувати _тільки_ з `@shared`. Віджети не повинні залежати один від одного або від сторінок.

    ⚠️ Використовувати тільки міксини в pug для нових віджетів!

4.  **`shared/`**
    - Містить **базові перевікористовувані елементи**, які не мають прив'язки до бізнес-логіки.
    - Сюди входять: UI-kit (кнопки, інпути), утиліти, константи, міксіни, глобальні іконки.
    - **Правила імпорту:** Цей шар не може імпортувати нічого з верхніх рівнів (`widgets`, `pages`, `app`).

    ⚠️ Використовувати тільки міксини в pug для нових ui компонентів в папці src/shared/ui!

## 📦 Імпорти та Аліаси

Для зручності в проекті налаштовані шляхи (aliases).

### JavaScript / SCSS

У файлах `.js` та `.scss` використовуйте аліаси:

- `@app` → `src/app`
- `@pages` → `src/pages`
- `@widgets` → `src/widgets`
- `@shared` → `src/shared`

**Приклад (JS):**

```javascript
import { MyButton } from "@shared/ui/button";
import { Header } from "@widgets/header";
```

### Іконки

Щоб додати нову іконку в спрайт її потрібно закинути в папку src/shared/icons/

Використання в pug

```
svg
    use(href="./icons.svg#SvgName")
```

ID це - назва файлу в стилі PascalCase

⚠️ Для того щоб нова svg відображалось на сайті потрібно відкрити сторінку в новій вкладці. Я поки що не знаю як це пофіксити. ⚠️

### Створення нової сторінки
